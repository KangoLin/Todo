# 土豆Todo — 项目调研总结报告

> 离线优先的 Trello 替代品 — 原生桌面看板工具  
> 技术栈：Tauri v2 + React 19 + TypeScript + Rust + SQLite  
> 项目代号：土豆Todo

---

## 一、项目定位

**一句话定义：** 一款离线优先、原生性能、MIT 纯净协议的看板（Kanban）项目管理桌面应用，面向个人用户和小团队，未来扩展到移动端。

**核心差异化：**
- ✅ 离线可用 — 数据存本地 SQLite，断网也能用
- ✅ 原生性能 — Tauri v2，安装包 <10MB，非 Electron 套壳
- ✅ 数据私有 — 数据完全归用户，不上传云端（云同步为可选增值）
- ✅ 买断制 — 一次付费，永久使用，非订阅
- ✅ 轻量聚焦 — 只做看板核心体验，不塞无关功能

## 三、中国应用商店评论调研

### 3.1 市场背景

- **Trello 在国内被墙**，用户无法正常访问，这是国内市场的最大特殊性
- 国产替代品应运而生，但**全部是纯云 SaaS，无一支持离线**
- 国内用户对订阅制接受度更低，偏好买断制

### 3.3 中国用户特有痛点

| 痛点 | 说明 |
|------|------|
| **Trello 被墙** | "每次用 Trello 都得翻墙"是最常见开场白 |
| **飞书/钉钉生态绑定** | 公司强制使用某生态，工具选择受限 |
| **"大而全"疲劳** | 国内工具倾向不断加功能，变得臃肿 |
| **订阅反感** | 国内用户更倾向一次付费买断 |
| **数据隐私焦虑** | "数据在阿里/腾讯服务器上不放心" |
| **与微信集成** | 国内办公通信以微信/企微为主 |
| **中文字体渲染** | Electron 下中文渲染模糊是常见抱怨 |

### 3.4 国内应用市场上架要求（重要）

| 要求项 | 详情 | 费用估算 | 周期 |
|--------|------|----------|------|
| **公司主体** | 华为/小米/Vivo/Oppo 不再接受个人开发者 | 代理注册 ¥2,000-3,500 | 1-2 周 |
| **软著**（软件著作权） | 所有安卓市场硬性要求 | 自己办免费 / 加急 ¥600-3,000 | 普通 2-3 月 / 加急 3-30 天 |
| **ICP 备案** | 联网应用必须完成 | ¥500+/年（含服务器） | ~20 天 |
| **App 联网备案** | 2025-2026 新增要求 | 免费 | ~20 天 |
| **对公账户** | 市场认证验证身份 | ¥500-1,200/年 | 1-2 周 |
| **各平台审核** | 各市场有独立审核标准 | 免费 | 1-7 天/平台 |

**各市场流量排名：**
1. **华为应用市场** — 自然量最多，出海能力最强 ✅
2. **小米应用商店** — 量大但偏大厂
3. **Vivo / Oppo 商店** — 量可观
4. **应用宝（腾讯）** — 量一般，微信生态有优势
5. **酷安** — 技术用户集中，开发者友好，**推荐早期冷启动**
6. **百度手机助手 / 360 / 阿里** — 已边缘化

**策略建议：** 早期先上酷安 + 官网直接下载，待完成软著和公司注册后再铺全渠道。

## 五、技术栈决策

| 层次 | 选型 | 版本 | 理由 |
|------|------|------|------|
| **桌面外壳** | Tauri | v2 | 原生性能（WebView 而非 Chromium），<10MB 安装包，支持 macOS/Windows/Linux/iOS/Android |
| **前端框架** | React | 19 | 生态最成熟、Hooks 架构清晰、SSR 支持 |
| **类型系统** | TypeScript | 5.x | 类型安全、IDE 支持好 |
| **后端语言** | Rust | latest | 原生性能、内存安全、与 Tauri 原生绑定 |
| **数据库** | SQLite (via rusqlite) | latest | 零配置、单文件、WAL 模式高并发、FTS5 全文搜索 |
| **状态管理** | Zustand | 5.x | 极简 API（~1KB），无 boilerplate，Zustand v5 支持 Signals |
| **服务端状态** | TanStack React Query | 5.x | 缓存/同步/去重，适配本地 SQLite 查询模式 |
| **拖拽** | Atlassian Pragmatic DnD | latest | Jira 同款，流畅度优于 @dnd-kit，无障碍支持完善 |
| **富文本** | Tiptap | latest | 基于 ProseMirror，JSON 格式存储，支持代码块/链接/任务列表 |

| **包管理** | pnpm | latest | 速度快、节省磁盘、严格依赖管理 |

## 七、功能路线图

### Phase 1 — MVP 看板核心（目标：可用的最小产品）

- 项目管理（创建/编辑/删除/重命名项目，侧边栏导航）
- 看板管理（多看板切换，创建/编辑/删除看板）
- 列管理（创建/编辑/删除列，拖拽重新排序）
- 卡片 CRUD（标题、描述、截止日期、优先级、颜色标记）
- 卡片拖拽（跨列移动、列内排序，实时 SQLite 写入持久化）
- 子任务（创建/勾选/排序）
- 标签系统（创建标签、颜色管理、按标签筛选）
- FTS5 全文搜索（标题+描述实时搜索）
- 本地 SQLite 存储（WAL 模式，自动备份）
- 基础 UI 框架（Tailwind v4 + shadcn/ui，浅色/深色主题）
- 键盘快捷键（方向键导航、快速创建、搜索 Cmd+K）
- 基础数据导出（JSON 导出/导入）

### Phase 2 — 增强功能（目标：差异化竞争力）

- 表格视图（类似电子表格的行列编辑模式）
- 统一看板（跨项目卡片的总览视图）
- 仪表盘（统计图表：各列卡片数、逾期卡片、标签分布）
- 多列视图（日历视图、时间线/甘特图视图）
- WIP 限制（列级上限，超出时视觉警告/禁止拖入）
- 看板模板（预设模板：Scrum/Kanban/个人/销售 Pipeline）
- 自动备份（定时备份 + 手动备份 + 备份管理）
- 卡片附件（拖拽文件到卡片，存储在本地）
- 看板分享（只读或编辑权限的本地网络共享）
- 数据统计（生产力报告、完成周期时间）

### Phase 3 — 云同步 & 移动端（目标：全平台覆盖）

- 可选云同步（端到端加密，用户可选择纯本地或开启同步）
- 同步后端（Rust 编写的轻量同步服务器，可选自托管）
- 冲突解决（基于 LWW 或 CRDT 的冲突合并策略）

## 九、关键决策与结论

### 9.1 关键决策记录

| 决策 | 结论 | 理由 |
|------|------|------|
| **协议策略** | 从零自研，MIT 发布，不复制任何 GPL/Fair-Use 代码 | 确保应用商店合规销售 |
| **架构选型** | Tauri v2 而非 Electron | 原生性能 <10MB，未来移动端支持 |
| **数据库** | SQLite 而非 IndexedDB 或第三方云 DB | 完全离线，FTS5 搜索，WAL 模式并发 |
| **拖拽库** | Atlassian Pragmatic DnD 而非 @dnd-kit | Jira 同款，更流畅，无障碍更好 |
| **参考项目** | Kabin（chlee1001/kabin）作为架构参考 | MIT 协议，Tauri v2 + React 19 + SQLite，理念高度吻合 |
| **排除项目** | Planka | Fair-Use License 禁止商用 |
| **首发市场** | 海外优先（验证产品）→ 国内（合规就绪后） | 国内上架门槛高（软著+公司+备案），先海外快速验证 |
| **定价模式** | 买断制 + 可选云同步订阅 | 迎合用户偏好，避免订阅疲劳 |
| **功能哲学** | 克制添加，只做看板核心 | 不做第二个 Jira/ClickUp |

### 9.2 核心优势总结

```
土豆Todo vs. 竞品矩阵

                    Trello  Planka  Focalboard  Kanboard  Kabin  土豆Todo
离线可用              ❌      ❌       ❌         ❌       ✅       ✅
原生 <10MB           ❌      ❌       ❌         ❌       ✅       ✅
MIT 纯净协议          ❌      ❌       ✅         ✅       ✅       ✅
商用合规              ✅      ❌       ✅         ✅       ✅       ✅
买断非订阅            ❌      ❌       ✅         ✅       ❌       ✅
轻量聚焦              ✅      ✅       ❌         ✅       ✅       ✅
现代化 UI             ✅      ✅       ✅         ❌       ✅       ✅
未来移动端            ✅      ❌       ❌         ❌       ❌       ✅
```

**蓝海定位：** 市场上存在且只存在一个空白位置 — "离线优先 + 原生桌面 + MIT 商用合规 + 买断制 + 轻量聚焦"。这就是土豆Todo 要占据的位置。

### 9.3 风险与缓解

| 风险 | 可能性 | 缓解措施 |
|------|--------|----------|
| 市场已有类似产品（如 Stack/Brisqi） | 中 | 差异化在于 MIT 开源 + 买断更低价格 + 移动端支持 |
| Tauri v2 生态不够成熟 | 低 | 已有多款 iOS App Store 成功案例验证 |
| SQLite 并发性能瓶颈 | 低 | WAL 模式 + 合理的事务管理，个人使用场景远低于瓶颈 |
| 用户对买断制长期收入存疑 | 中 | 云同步订阅 + 商业许可（企业付费）作为长尾收入 |
| 国内上架合规周期长 | 高 | 先海外验证，国内走加急软著 + 代理注册公司并行 |
